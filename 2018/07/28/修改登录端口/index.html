自建vps修改22端口

发表于 2018-07-28 | 分类于 linux之海 | | 阅读次数:

当你租了台服务器，准备大展拳脚的时候。需要注意的一点是我们linux服务器默认就是22的端口,很多破解登录的脚本会疯狂扫描。就算你设置的密码足够复杂，可是也不想看到日志文件被这种垃圾信息塞满吧。
下面我教大家怎么处理

  1. 先用命令查询日志
    cat /var/log/secure | awk '/Failed/{print $(NF-3)}' | sort | uniq -c | awk '{print $2" = "$1;}'
    如果发现很多数据, 说明很多软件在登录破解你的服务器, 如果没有错误数据, 那倒问题不大,但是为了服务器的安全和资源占用，还是修改端口的好
  2. 修改22端口

    修改ssh的配置文件 sudo　vim /etc/ssh/sshd_config 找到port=22这行并在下面添加一行port = 12222 (端口自拟，还是推荐10000以上的端口难以被发现) 然后:wq保存退出
    重启ssh服务
    /etc/init.d/sshd restart
    这样ssh端口将同时在22和12222上运行，注意在防火墙中放开12222端口
    如果和我一样用的xshell的或者其他连接工具的话可以重新开个连接用12222端口测试下，成功登录后将22端口注释掉.
  3. 修改services文件（选用）
    如果是和我一样还要使用ssh 进行端口转发，内网映射等功能的话，还是要修改下services文件

    sudo vim /etc/services
    然后找到ssh对应的端口号22， 还是一样复制端口为22的那行，然后将其改成12222，注释掉ssh 22的那行，就可以正常的使用了

服务器登录端口的修改只是对那些疯狂的脚本扫描做出的一点防护，要是别人针对到你的端口进行恶意登录套密码，就算密码设置的再复杂，看到/usr/log/secure里面的登录记录也是很烦的，那么要怎么样防止这个针对的恶意扫描呢，再有时间再写（挖坑x1）
