使用有道云进行国际化配置

发表于 2018-07-29 | 分类于 菜鸡java | | 阅读次数:

之前需要国际化，需要翻译文本量有点大，于是写了个专门转国际化（中文properties转英文properties）的项目。果然人类惰性推动了社会的发展。

  1. 准备好有道云账号
    为什么选择了有道云，这个主要的原因是有道云字典是我长期使用的翻译软件，如果你们想
    用其他的sdk如 百度的或者google的，自信去官网查看sdk的使用方式，我这里只针对有道云sdk进行讲述。
    • 打开https://ai.youdao.com/index.s选择登录有道智云，然后如果你没有账号的话注册一个也行，注册会送100元的代金卷，用来进行后续的翻译api调用是完全没有问题了。
    • 登录有道智云，获取appid和key，这个就很简单了，创建应用，点击我的应用，对应的appid和key都能看见，
  2. 建立调用程序
    • 获取本地的中文properties和英文properties，这个获取有很多种方式了，可以自己写一个properties的读取utils进行获取，我就是采用的这个方式，注意这里有个坑，就是使用propries.load进行读写的时候是hash读写，不会按照顺序进行文件从上往下进行翻译，这里我采用的方式是继承Properties类，在Properties类进行readline的时候存key


然后在初始化的时候 new OrderdProperties();

- 调试sdk 有道智云上面会提供一个有道云的api调用地址
`http://openapi.youdao.com/api`,

我将有道的配置写入了applications.properties中以便于修改和调用，我使用的是baidu出来的httputil文件中的post请求。测试通过后就面临下一个问题

- 调用线程池 在一个大的properties文件中不可能同时将所有的需要翻译的文本传过去，只能一条条进行翻译，不然全部传过去可能会翻译出错误，所以这里就考虑到一个并发的问题了，我这里的做法是开启了线程池


比较偷懒的是采用的是定长线程池而不是自己写的线程池，根据阿里巴巴的代码规范还是要自己定义的线程池为好，我这里抽象的也很菜。回头看果然自己的代码最辣眼睛。

  1. 文件读写
    也是同样设置好中文和英文properties的文件地址就行，不过我是用的springboot来写的这个项目，所以文件的资源路径变成了target下面的路径，导致翻译没有东西出来，后来写了个方法强制指定到本地文件的properties中。
    `java
    private static String getLocalPath(String path){
    path=fileUtil.class.getClassLoader().getResource(path).getPath();
return path.replace("target/classes","src/main/resources");
    }`
    这个项目最大的坑是我写的j2ee项目，本想着以后可以通过页面读取文件-返回下载文件的方式进行翻译，我还是高估了我自己，实在太懒了，方法写出来能用就用了写啥子的页面吧。导致这个项目最令人诟病的地方-它是直接写到了application.main方法启动的（因为我使用了一些springcontext里面的方法。。）然后我还注册到了eureka里面（想太多了api接口都不想写完,本地调用就完事了）。我试了试效果还行。翻译速度在100行以内都挺快
