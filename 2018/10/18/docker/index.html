docker部署springboot项目

发表于 2018-10-18 | 分类于 运维 | | 阅读次数:

最近微服务化应该是比较火了，docker化springboot项目应该是其中基础了，来写一写docker相关的东西吧。

dockerfile的编写

一个典型的dockerfile文件包含的几个部分

  • from 引入镜像
  • volume 挂载文件
  • copy 复制文件
  • entypoint 入口执行
  • expose 端口映射
    还有其他的指令可以在https://yeasy.gitbooks.io/docker_practice/image/dockerfile/中找到，要是想建一个快速的springboot项目，我提到的几个是比较关键的参数，上面我提出来的例子中是没有端口映射这个参数的，因为我是使用的docker-compose.yml的方式进行的docker启动。有些部分是直接写到了yml中。

docker启动

启动就很简单了,我总结出3种方式可以启动

  1. 使用docker-maven-plugin的一个插件进行启动，需要修改maven pom文件。

这样的好处是少一个docker的build过程，放在docker环境下，直接运行mvn clean package(or install?) 的话是可以直接生成镜像的(实际上我运行的是 mvn clean package docker：build)，这个方式可以在插件这里设置很多参数

  1. 直接使用 docker build构建,我上面的例子就是直接构建的例子，将生成好的jar包放到dockerfile同级目录，然后使用docker build直接进行构建
  2. 使用docker-compose.yml进行构建，这样做主要是为了同时运行多个docker项目，方便一键化部署。附上我的docker-compose文件

可以看出端口是在这个文件中以ports的形式进行映射，

docker 更新

讲到docker第一印象就是它的便利性吧，什么一键化部署，集群的话很是方便，主要是它从云端存取镜像，但是国内的网站访问dockerhub不能…，这就很尴尬，国内公司一般都会有他自己的docker仓库，这又和我以前说到的nexus仓库有着紧密的联系了，个人开发者的话建议科学上网啥的弄下，dockerhub还是比较关键。虽然不能上dockerhub，但是并不影响它的下载。springboot项目更新的话我推荐使用第二种形式，方便快捷。

总结

现在的环境对于开发者来说确实要求比较多了吧，从前端到后台到运维什么都要会一点。但是这个也是阻挡不了我走全栈开发的路的，加油
